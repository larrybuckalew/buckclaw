How It Works:
-------------
1. Main agent calls subagent_dispatch(agent_id="cookie", task="...", context="...")
2. Tool creates new session in history_store
3. Tool subscribes temporary handler to DispatchResultEvent (filtered by session_id)
4. Tool publishes DispatchEvent(session_id=new_session, parent_session_id=trace_info)
5. AgentWorker.handle_dispatch() receives event
6. Worker loads Cookie agent and runs session.chat(task)
7. Cookie responds and publishes DispatchResultEvent
8. Handler resolves future with response
9. Tool unsubscribes and returns JSON: {"result": "...", "session_id": "..."}
10. Main agent presents result to user

Critical Design Points:
-----------------------
- Subscribe BEFORE publish to prevent race conditions
- Tool returns None if agent has no peers (no dispatch targets)
- Agent cannot dispatch to itself
- EventBus keeps agents decoupled (no direct imports)
- parent_session_id enables tracing of call lineage
- Dynamic tool description lists available agents as XML
